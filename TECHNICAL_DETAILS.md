# DSS Web - Technical Details

## Architecture Overview
The Digital Service Standard (DSS) portal operates as a Static Site Application (SPA) with a decoupled data preparation pipeline. It requires no active backend server or runtime database, relying instead on pre-computed JSON files generated by AI and Node.js scripts.

## Tech Stack
- **Frontend Framework:** React 18, Vite
- **Routing:** React Router DOM (v7)
- **Styling:** Vanilla CSS, CSS Variables for theming, Lucide React for iconography.
- **Data Pipeline Scripts:** Python (Pandas, Google GenAI SDK), Node.js (fs, path).
- **Deployment:** GitHub Pages (via `gh-pages` npm package).

## Data Processing Pipeline

1. **Raw Data Ingestion (`DSS_Controls.xlsx`)**
   The primary source of truth is an Excel spreadsheet containing the raw, formal controls, their IDs, compliance impact levels, and other metadata.

2. **AI Refinement (`process_controls.py`)**
   A Python script reads the Excel file and passes each control's data to the Google Gemini API (gemini-2.5-pro model). The AI generates improved, readable Thai content, breaking it down into:
   - Recommendations
   - Checklists
   - Evidence requirements
   The output is saved as individual JSON files (e.g., `public/controls/AI-02.json`) within the web application's public directory.

3. **Index Generation (`scripts/generate_index.js`)**
   To support multidimensional filtering and search without a database, a Node.js script recursively scans the `public/controls` directory, extracting both the AI-refined content and critical metadata (Compliance, Service Type, Impact Levels). It generates a monolithic `index.json` file.
   - *Automation:* This script is hooked into the `package.json` build lifecycle using `predev` and `prebuild` scripts, ensuring the local server and production builds always use the latest index data.

## Frontend Implementation Details

- **Home Page (`src/pages/Home.jsx`)**
  - Fetches the `index.json` once on component mount.
  - Uses `useMemo` hooks to dynamically calculate available filter options (e.g., unique sets of Service Types).
  - Implements a cascading filter pattern where the primary data array is reduced based on search text and dropdown selections *before* being grouped by category.

- **Control Detail Page (`src/pages/ControlDetail.jsx`)**
  - Uses the URL parameter (`/:id`) to dynamically fetch the specific full JSON file for a control (e.g., `/controls/AI-02.json`).
  - Renders the robust AI-generated data (Checklists, Evidence) in an interactive UI (glassmorphism design).

## State Management and Routing
- Complex global state management (like Redux) is avoided due to the read-only nature of the application. Local component state (`useState`) handles search and filtering.
- Client-side routing is configured using Vite's integrated server configuration to redirect all paths to `index.html`, letting React Router handle dynamic page transitions smoothly.
